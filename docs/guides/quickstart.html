Quickstart

Pull your board, create a task, move it across the workflow — all in under a minute. By the end of this guide you'll have a working API key and a feel for how Elevate's REST surface fits together.

1. Generate an API key

Open the Elevate app and head to Settings → API Keys. Hit Generate API Key, give it a memorable name (something like local-dev or my-first-bot), and pick the Read & Write scope so you can both fetch and modify data.

Copy the key the moment it appears. We only show the raw value once — after you close the dialog it's gone for good and you'll need to make a new one.

Treat your API key like a password.Keys with the Elevate_ prefix grant full access to whatever scope you set. Keep them out of version control, commit history, and client-side bundles. Environment variables or a secrets manager are the right home for them.

2. Make your first request

The fastest way to get a feel for the API is to fetch your full board state. One GET, one header, and you'll see every column and task in your workspace come back as JSON.

Replace trust-and-safety with your own workspace ID (you can find it in the URL when you're viewing a board) and paste in the key you just generated:

GET/board/{workspaceId}
curl https://app.arcnvd.com/api/v1/board/trust-and-safety \
  -H "X-API-Key: Elevate_your_key_here"

You'll get back the whole board in one shot: every task, every column, the column ordering, and metadata like priorities, assignees, due dates, tags, comments, attachments, and checklists. Most integrations start here — pull once, then issue smaller writes.

3. Create a task

Tasks always live inside a column, so the first thing you need is a columnId from that board response. Pick any column you like (typically the leftmost "To do" or "Backlog") and post a new task to it:

POST/tasks
curl -X POST https://app.arcnvd.com/api/v1/tasks \
  -H "X-API-Key: Elevate_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"columnId":"col-1","title":"Fix login bug","priority":"High","tags":["bug"]}'

Only columnId and title are required. Everything else — priority, tags, description, due date, assignee — is optional and can be filled in later. The response gives you the new task's full record, including its generated id and short code, which you'll use for follow-up calls.

4. Move a task

This is where Elevate earns its keep. Moving a task between columns is how you reflect progress through your workflow — "In progress", "Review", "Done" — and it's one of the most common API calls you'll make.

Use the task id from the previous step, and pick a different column for targetColumnId:

PATCH/tasks/{taskId}/move
curl -X PATCH https://app.arcnvd.com/api/v1/tasks/task-1/move \
  -H "X-API-Key: Elevate_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"targetColumnId":"col-2"}'

Behind the scenes Elevate records this as a task.moved activity event and fires the matching webhook to anyone subscribed. If you have a Slack bot, a Discord notifier, or a CI pipeline tied into Elevate, this single call is what wakes them up.

Next steps

That's the whole loop. From here you can keep building in two directions:

  • Build out the integration. Read the Authentication guide to compare JWT bearer tokens (good for browser clients) with API keys (better for bots and CI), or browse the full API reference for every endpoint with parameters, schemas, and examples.
  • Make it real-time. Polling works, but webhooks are nicer. The Webhooks guide shows how to subscribe to events, verify the signature on incoming payloads, and handle retries when your endpoint blips.

If you get stuck, the API reference is the source of truth — every operation listed there is generated directly from the running backend, so it can't go stale.
